Is your codebase rock SOLID

2018-04-18

Introduction

Anyone can write code, but to write quality code that scales requires one to apply disciplined practices. As software developers our ultimate goal is to create software that is

  • Tolerant to change
  • Easy to understand
  • The basis of components that will be used in other systems.

I introduce you to the SOLID principles that were introduced by Robert C Martin aka Uncle Bob in 1995 (but still very much relavant today). These principles are going to solve a lot of hells that arise due to spagghetti code and make the future you thank you for the code you are writting now.

So what then are the SOLID principles

SOLID is an acronym for

  • S - The Single Responsibility Principle (SRP)

    Each class should have one and only one reason to change

  • O - The Open-Closed Principle (OCP)

    Classes should be open for extension and closed for modification.

  • L - The Liskov Substitution Principle (LSP)

    Derived classes must be substitutable for their base classes

  • I - The Interfacer segregation Principe (ISP)

    Classes should not depend on code they do not use

  • D - The Dependancy inversion principle (DIP)

    Depend on abstractions and not on conctretions

Liskov Substitution Principle

Derived classes must be substitutable for their base classes. Which simply means if class B is derived from class A, and class C is also derived dfrom class A then it is possible to derive class C class B without any problem.

Another way to say it would be: classes that reference base classes must be able to use objects of the derived class without knowing it.

Now that you have an understanding of the SOLID principles you could use it to refactor your codebases to achive the following:

  • Flexibility of code
  • Orient your code towards future use rather than just immediate needs
  • Produce modular code that is reausable even out of original context
  • Produce code that is easy to test
  • Develop systems that are resilient to changes
  • Developer joy - SOLID principles results in cleaner code because of the extra effort it needs and spaghetti free code contributes significantly to developer joy and motivation
